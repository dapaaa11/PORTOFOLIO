# Additional AI Agent Operational Rules

# Purpose

This document defines operational rules and behavioral constraints for all AI coding agents working on the Dava Ardana Portfolio project.

The purpose of these rules is to:

* maintain architecture consistency
* preserve design direction
* protect performance quality
* prevent over-engineering
* maintain clean development workflows

All AI agents must treat this document as a core development reference.

---

# GitHub Workflow Rules

# Mandatory Progress Push

AI agents MUST push every meaningful progress update to GitHub.

This ensures:

* version safety
* progress tracking
* rollback capability
* stable development history

---

# Push Frequency

AI agents should push:

* after completing a feature
* after fixing a bug
* after finishing a section
* after successful testing
* before large refactors

Avoid:

* giant untracked changes
* massive single commits
* long uncommitted sessions

---

# Commit Message Rules

Commit messages must be:

* clean
* descriptive
* short
* professional
